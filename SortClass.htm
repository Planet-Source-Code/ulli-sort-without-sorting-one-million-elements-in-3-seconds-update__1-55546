Overview

 

This Sort-Class can be used to retrieve the elements in a one-dimensional table of strings in either ascending or descending sequence. The table itself is not altered in any way by this process, rather pointers into the table are returned which point to the elements in the table in the requested order. The range of subsripts or indexes (dimension) for the table may unbiased (1 to n) or may be biased by any amount but must not include zero (see LowBound and HighBound property). The Sort is stable, i.e. equal keys are returned in their original (positional) sequence. This feature can be used to sort a table on more than one key progressing from the least significant to the most significant key.

 

An ID can be assigned to the SortTable method and the SortIndirect method, so that the application can determine during the NextPointer event which Sort method is being executed.

 

Tests on a 1800 MHz Athlon processor have shown that the Sort can fire the NextPointer Event every 2.5 microseconds average  (sorting on a five byte random key) when the  called event routine itself executes no code, sorting 100,000 elements in under 0.3 seconds. The speed varies only very slightly with the number of elements to be sorted, so one million elements take about three seconds. Any special presorting has no measurable effect on speed (Quicksort by contrast is almost killed by a presorted sequence).

 

The algorithm used in this sort builds linked lists and then linked sublists within lists recursively progressing until the final list structure represents the sorted result.

 

The Sort will temporarily allocate memory, the size being dependent on the amount of data to be sorted and the KeySize (four bytes per element plus KeySize kilobytes). Also some stack must be available and the amount depends on the key value distribution.

 

 

Properties, Methods and Events

 

 

Alphabet Property

Sets / returns the collating sequence for the Sort. Characters not in Alphabet are placed in their native collating sequence after those in Alphabet. Interacts with the KeyTranslation property.

 

Syntax

Sort.Alphabet = value As String

 

Remarks

This property governs the collating sequence for the Sort. It overrides and is overridden by the KeyTranslation property..

 

Data Type

String

 

 

HighBound Property

Sets / returns the upper boundary of the sort range.

 

Syntax

Sort.HighBound = value As Long

 

Remarks

The LowBound property and the HighBound property must both be negative or both be positive.

 

Data Type

Long

 

 

KeyPosition Property

Sets / returns the starting byte position for the sort key  (counting from 1).

 

Syntax

Sort.KeyPosition = value As Long

 

Remarks

When RightToLeft is False then KeyPosition + KeySize - 1 must not be greater than the table element size if that part of the key is needed to determine uniqueness or sequence if the PartialKeys property is set to NotAllowed.

 

When RightToLeft is True then KeyPosition - KeySize  must not be less than zero if that part of the key is needed to determine uniqueness or sequence if the PartialKeys property is set to NotAllowed.

 

Data Type

Long

 

 

KeySize Property

Sets / returns the length of the sort key in bytes.

 

Syntax

Sort.KeySize = value As Long

 

Remarks

KeySize must not be less than 1.

 

Data Type

Long

 

 

KeyTranslation Property

Used to perform a sort key translation.  Interacts with the Alphabet property.

 

Syntax    

Sort.KeyTranslation = value As String

 

Remarks   

The Ascii value + 1 of each byte in the original keys is used to address a byte in this string. The byte thus found is then (for comparison and sorting purposes) substituted for the original byte. This property string can therefore only be 256 or zero bytes long, the latter being the value to switch key translation off. The following code fragment is used internally to achieve key translation:

 

SortKey_Character = Mid$(KeyTranslation_Character, Asc(OriginalKey_Character) + 1, 1)

 

Therefore this property must contain a string of 256 bytes. This property overrides and is overridden by the Alphabet property.

 

Data Type

String

 

 

LowBound Property

Sets / returns the lower boundary of the sort range.

 

Syntax

Sort.LowBound = value As Long

 

Remarks   

The LowBound property and the HighBound property must both be negative or both be positive and within the table bounds and HighBound must not be less than LowBound.

 

Example  

Assuming the following table definition:

 

      Dim Table(-20 To 20)

 

     Okay                                       Rejected

 

      LowBound = 1           LowBound = -10

      HighBound = 10         HighBound = 10

 

      LowBound = -10         LowBound = 10

      HighBound = -2         HighBound = 5

 

                                LowBound = 1

                                HighBound = 30

 

Data Type

Long

 

 

NextPointer Event

Fired when the next element in sort sequence has been determined.

 

Syntax

Private Sub Sort_NextPointer(ByVal SortId As Long, ByVal Pointer As Long, Cancel As Boolean)

 

Remarks

This event is fired once for each element in the sorted range. The Pointer points to the next element in either ascending or descending key sequence depending on the setting of the SortDirection property.

 

The Cancel argument may be set to True at any time to terminate a Sort prematurely.

 

 

PartialKeys Property

Defines the ordinal position of missing characters in partial keys.

 

Syntax

Sort.PartialKeys = PartialKeys

 

Remarks

The Sort examines the keys byte by byte and whenever a byte required to determine uniqueness or sequence is not contained within the sort-element then this property is referred to and governs the Sort's behavior. If uniqueness or sequence can be determined from a partial key that is contained within a sort-element then this property is not used.

 

Example

With a KeyPosition = 1 and a KeySize = 3 assuming the following original sequence:

 

      Original Sequence       GreaterFullKeys        LessFullKeys

                              produces               produces

 

      A                       AA                      A

      AC                      AC                      AA

      BB                      A                       AC

      BBC                     BBB                     BB

      BBB                     BBC                     BBB

      AA                      BB                      BBC

 

NotAllowed will raise an error whenever a required key-byte is not contained within a sort-element. This would occur in the above example while trying to determine the sequence of A and AA or BB and BBB resp. but not with AA and AC because in this case the sequence can be determined without examining the missing third byte.

 

Data Type

PartialKeys (Enumeration)

 

 

QueryKey Event

Fired when a key is required during SortIndirect.

 

Syntax

Private Sub Sort_QueryKey(ByVal SortId As Long, ByVal Pointer As Long, Key As String)

 

Remarks

The key of the element pointed to by Pointer should be extracted and placed in the Key argument. KeyPosition, KeySize and RightToLeft are then applied internally to the Key argument to determine the exact position and length of the key.

 

 

RightToLeft Property

Defines whether the keys are evaluated from right to left end.

 

Syntax

Sort.RightToLeft = {True | False}

 

Remarks

Normally the key defined by KeySize and KeyPosition is evaluated from left to right; if however this property is set to True then the keys are compared starting at the right hand end. 

 

Data Type

Boolean

 

 

SortDirection Property

Sets / returns the requested sort sequence.

 

Syntax

Sort.SortDirection = SortDirection

 

Remarks

The Sort is stable, i.e. elements with equal keys are pointed to (during the NextPointer event) in their original (positional) sequence.

 

Data Type

SortDirection (Enumeration)

 

 

SortIndirect Method

Performs the Sort and fires NextPointer events. Keys are queried via QueryKey events.

 

Syntax

Number = Sort.SortIndirect(SortId) As Long

 

Remarks

This function returns the number of distinct keys that have been sorted.

 

 

SortTable Method

Performs the Sort and fires NextPointer events. Keys are taken from elements in the Table.

 

Syntax

Number = Sort.SortTable(Table As OLEType8200, SortId) As Long

 

Remarks

This function returns the number of distinct keys that have been sorted.
